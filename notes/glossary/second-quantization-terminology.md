# Second-quantization terminology

This glossary records the preferred physical terminology and its correspondence with
project-specific Lean identifiers. Physics prose names the physical operation first; a Lean
identifier is given in backticks only when the implementation matters.

| Source | Purpose | Concrete referent | Role | Before/after relation | Candidate term | First-use definition |
|---|---|---|---|---|---|---|
| `SecondQuantization/Common/ExchangeCommutator.lean` | Reorder two linear maps with a statistics sign | `A.comp B - ζ • B.comp A` | value | Signed difference of two compositions | `ζ`-commutator / exchange commutator | For linear maps `A,B`, define `[A,B]ζ := A.comp B - ζ • B.comp A`; this is not the graded commutator and not a thermal contraction. |
| `SecondQuantization/Common/TimeOrdering.lean` | Order two time-labelled operator factors | Piecewise linear-map expression selected by `τA` and `τB` | means | Swapping time order changes composition and may multiply by the statistics sign | imaginary-time-ordered product `Tτ` | `Tτ` orders factors by imaginary time; the operation itself is not thermal until placed inside a thermal expectation. |
| `SecondQuantization/Common/BlochDeDominicisPairing.lean` | Represent a complete matching of operator positions | Fixed-point-free involutive permutation of `Fin (2*n)` | state | Each position maps to its unique paired position | perfect pairing of operator positions | A pairing is a perfect matching of the finite operator slots; it is combinatorial and carries no expectation value by itself. |
| `SecondQuantization/Common/BlochDeDominicisPairing.lean` | Remove the pair containing the first operator | Delete positions `0` and `partner 0`, then transport the restricted pairing | event | A pairing on `2*(n+1)` slots becomes one on `2*n` reindexed slots | pair deletion/reindexing (Lean: `eraseZeroPair`) | This is an induction operation on finite pairings, not a physical process or a thermal operation. |
| `SecondQuantization/Common/BlochDeDominicisPairing.lean` | Determine the statistics-dependent pairing factor | Crossing count and `ζ ^ crossingCount` | value | Pairing geometry determines the bosonic/fermionic weight | crossing number and statistics weight | Count interleavings `a < c < b < d`; use `ζ = +1` for bosons and `ζ = -1` for fermions. |
| `SecondQuantization/Fermionic/ImaginaryTimeEvolution.lean` | Realize free evolution on the algebraic basis | `Common.diagonalEvolution (fermionEnergy ε) τ` | value | Basis vector is multiplied by a scalar exponential | algebraic basis-diagonal imaginary-time evolution | This is an algebraic realization on finite-support Fock vectors, not an operator exponential on a completed Hilbert space. |
| `SecondQuantization/Fermionic/WeightedDiagonalFunctional.lean` | Normalize a weighted diagonal sum | `weightedTrace w A / weightSum w` for arbitrary complex `w` | value | Weighting followed by normalization | normalized weighted diagonal functional | Call it thermal/Gibbs only after specializing `w` to a positive Boltzmann weight and proving the required normalization facts. |
| `SecondQuantization/Fermionic/WeightedDiagonalFunctional.lean` | Sum an arbitrary finite weight over occupation configurations | `∑ n, w n` for `w : FermionOccupation Mode → ℂ` | value | An arbitrary weight sum is later specialized to the free Gibbs partition function | total weight (`weightSum`); partition function only after Gibbs specialization | `weightSum w` denotes the finite sum of an arbitrary weight. It is not itself a physical partition function. |
| `SecondQuantization/Fermionic/WeightedFreeTwoPointFunction.lean` | Form a two-operator correlator with free evolution and ordering | `weightedFreeTwoPointFunction ε w i j τ τ'` with arbitrary `w` | value | Ordered product is then evaluated by the weighted functional | weighted free two-point function; Gibbs Green function only for Gibbs specialization | The generic definition uses an arbitrary complex weight; reserve `G₀`/Gibbs terminology for the free Gibbs specialization. |
| `SecondQuantization/Fermionic/WeightedContraction.lean` | Evaluate a two-operator weighted diagonal functional | The free number-conserving Gibbs specialization gives a finite-temperature contraction | value | Functional evaluation, unlike an operator reordering identity | finite-temperature contraction / Bloch–de Dominicis contraction | A contraction is the c-number expectation of an ordered pair; the current generic lemmas are algebraic weighted-functionals and become thermal only after Gibbs specialization; do not use this name for `AB = 1 + ζ BA`. |
| `SecondQuantization/Fermionic/ImaginaryTimeOrdering.lean` and `Bosonic/ImaginaryTimeOrdering.lean` | Fix the statistics-specific ordering wrapper | `Common.timeOrderedProduct Statistics.* ...` | means | No state, weight, or β occurs in the definition | imaginary-time ordering | The wrapper is not thermal; the module name identifies the ordered imaginary-time operation. |
| `SecondQuantization/Common/QuantumLinkedCluster.lean` | Connect fermionic occupation moments to cumulants | Imports `Fermionic.WeightedDiagonalFunctional`; uses `FermionOccupation` and `FockSpaceFermionic` | module | Fermionic implementation of a Track-B bridge | fermionic linked-cluster bridge | This module remains a deliberately scoped fermionic bridge; its location is not changed in this rename PR. |
| `QuantumTheory/DensityOperator.lean` and `DensityOperatorTraceClass.lean` | Map a pure vector to its rank-one density operator | `ψ ↦ |ψ⟩⟨ψ|` | value | Pure vector representation becomes a density operator | pure-state density-operator embedding | “Purification” normally means a pure state on a larger space whose reduced state is a given mixed state; this definition is not purification. |
| `QuantumTheory/Entropy.lean` and `notes/model-and-assumptions.md` | Relate von Neumann entropy to thermodynamic entropy | Postulated identification `k_B S_vN = S_thermo` | relation | Mathematical entropy is connected to a thermodynamic quantity | Boltzmann's principle | In this project, “Boltzmann's principle” names the postulated identification of `k_B` times von Neumann entropy with thermodynamic entropy; the equality is not formalized. |
| `SecondQuantization/Common/OneParticleSpace.lean` | Provide a finite mode index type | Abstract `Mode : Type` with `modeCount`; no linear/Hilbert structure | type | Labels precede occupation states and Fock vectors | one-particle mode labels / mode index type | `Mode` is not yet a one-particle Hilbert space; do not call it `ℓ²(Mode)` until that structure exists. |
| `SecondQuantization/Fermionic/Occupation.lean` and `Bosonic/Occupation.lean` | Represent zero-occupation configurations | `∅` or `0` in the occupation configuration type | value | Configuration later maps to the Fock vacuum vector | vacuum occupation configuration | Reserve “vacuum vector/state” for `fockVacuum` (and ultimately a Hilbert-space state); these are configurations. |
| `SecondQuantization/Fermionic/FreeBoltzmannWeight.lean` | Assign free Gibbs weights | `exp (-β E(n))` for arbitrary real `β` | value | Weight becomes thermal only under ensemble assumptions | real inverse-temperature parameter / Boltzmann weight | Treat `β` as a real parameter in the algebraic theorem; impose physical `β > 0` or another ensemble condition where needed. |
| `notes/roadmaps/second-quantization.md` and `notes/model-and-assumptions.md` | Name the target expansion theorem | Finite-temperature quasifree expansion into pairings | purpose | Thermal expectation is expanded by pairings, unlike vacuum-only Wick statements | finite-temperature Bloch–de Dominicis theorem | Keep the name, but state the free/quasifree (Gaussian) Gibbs assumption explicitly; “thermal” alone does not imply a pairings-only expansion for an interacting state. |

## Accepted terminology in the current design

`Common/AlgebraicFock.lean` correctly treats `matrixCoeff`/`diagonalCoeff` as coordinate evaluations rather than Hilbert-space matrix elements. `FormalExp` and `FormalLogPartitionFunction` correctly distinguish formal series from analytic exponentials and from the future Dyson/linked-cluster theorem. `BlochDeDominicisPairing` already keeps pairings separate from contractions and expectations.
